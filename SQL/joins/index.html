    Joins

    Cartesian Join¶

    SELECT select_list
FROM table_a, table_b;

SELECT select_list
FROM table_a CROSS JOIN table_b;
-- both queries would return the same result.

    Notes

    • It is also known as Cartesian Product or Cross Join.
    • Replicates each row from the first table with every row from the second table. Creates a Join between tables by displaying every possible record combination.

    Examples

    1
2
    SELECT *
FROM oe.orders, oe.customers;
    Sources
    • Oracle Documentation - Cartesian Product

    Inner Join¶

    SELECT column_list
FROM table_a t1 [INNER] JOIN table_b t2
    USING (column_1 [, column_n]);

SELECT column_list
FROM table_a t1 [INNER] JOIN table_b t2
    ON (t1.column_1 = t2.column_1);

SELECT select_list
FROM table_a t1, table_b t2
WHERE t1.column_1 = t2.column_1;

    Notes

    • It is also known as Equijoin, Equality Join or Simple Join.
    • Creates a join by using a commonly named and defined column.
    • The USING clause specifies which columns to test for equality when two tables are joined.

    Examples

    1
2
3
    SELECT *
FROM oe.orders o JOIN oe.order_items oi
ON o.order_id = oi.order_id;
    Sources
    • Oracle Documentation - Equijoins

    Non-equality Join¶

    Joins tables when there are no equivalent rows in the tables to be joined;
    For example, to match values in one column of a table with a range of values in another table.
    Can be created by two methods:

    • Using the WHERE clause
    • Using the JOIN method with the JOIN … ON keywords

    Self-Join¶

    SELECT column_list
FROM table_a t1 [INNER] JOIN table_a t2
    USING (column_1 [, column_n]);

SELECT column_list
FROM table_a t1 [INNER] JOIN table_a t2
    ON (t1.column_1 = t2.column_1);

SELECT select_list
FROM table_a t1, table_a t2
WHERE t1.column_1 = t2.column_1;

    Notes

    • Joins a table to itself.
    • To correctly perform a Self-Join, both Tables must have an alias defined in the query.
    Sources
    • Oracle Documentation - Self Joins

    Outer Join¶

    -- LEFT OUTER JOIN: new syntax
SELECT column_list
FROM table_a t1 LEFT [OUTER] JOIN table_B t2
    ON (t1.column_1 = t2.column_1);

-- LEFT OUTER JOIN: old syntax
SELECT column_list
FROM table_a t1, table_B t2
WHERE t1.column_1 = t2.column_1(+);

-- RIGHT OUTER JOIN: new syntax
SELECT column_list
FROM table_a t1 RIGHT [OUTER] JOIN table_B t2
    ON (t1.column_1 = t2.column_1);

-- RIGHT OUTER JOIN: old syntax
SELECT column_list
FROM table_a t1, table_B t2
WHERE t1.column_1(+) = t2.column_1;

-- FULL OUTER JOIN: new syntax
SELECT column_list
FROM table_a t1 FULL OUTER JOIN table_B t2
    ON (t1.column_1 = t2.column_1);

-- FULL OUTER JOIN: old syntax
SELECT column_list
FROM table_a t1, table_B t2
WHERE t1.column_1(+) = t2.column_1(+);

    Notes

    • Includes records of a table in output when there's no matching record in the other table.
    Sources
    • Oracle Documentation - Outer Joins
